Survey

 

Survey Help

NOTE:  The Survey test case is intended to be run by a Microsoft onsite resource only.  This test case is used to process the results of the operational interview so that the data can be integrated into the formal SQLRAP report.  Customers should not run this test case.

 

 

The Survey test case integrates the results of the operational interview so that issues can be generated and included in the formal SQLRAP report.  The instructions below do not include the steps to perform the operational interview or process the data for inclusion into the SQLRAP Tool.  Refer to Appendix E of the Technical Reference – Internal Only version for detailed information on these processes.

 

 

Configuring the Survey Test Case

 

After the survey form has been filled, the results can be integrated into the SQLRAP tool so they show up as issues.

 

§       Copy the filled survey form from the engineer's computer to the customer's computer that has the SQLRAP tool installed. Do NOT copy it to the SQLRAP's Output folder.  Placing external XML files in this folder is not supported and can cause unexpected results.  The processed survey XML file can be copied to any other folder.

§       Open the Change Settings menu

§       Select the Survey test case

§       Change the "FilledOutSurveyFile" setting to the full path and file name in which the filled out survey file was copied. For example: C:\SQLRAP\Public\SQLRAP Survey.XML.

 

 

Running the Survey Test Case

§       Open the Run Tests screen

§       Select “Survey” and run it

 

NOTE:  The Survey test case results will appear in the View Results screen however it will be displayed in raw XML format only.  There is no way to view the operational survey in a user friendly format from within the SQLRAP tool.  Only the issues from the operational survey are displayed in a readable format.

 

 

View Survey Issues

After the Survey test case has been run, the results will be made into issues. Use the Issues Viewer to view them. The survey-related issues will appear under the following categories:

            - Backup

                        - Strategy and Planning

                        - Backup and Restore

            - Other

                        - Environmental Dependencies

            - Operational Excellence

                        - Change Configuration and Release

                        - General

                        - Monitoring

                        - Service Level Management

 

 

Annotating Survey Issues

§       By default, all survey issues are marked as report visible.

§       A Notes section is available to add custom comments. Use this to expand on why certain answers were given.  For example, it may be necessary to explain that two groups had different responses to the same question.

§       Survey issues do not contain Troubleshooting Steps or Resolution sections.
