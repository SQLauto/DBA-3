MAXDOP

 

The max degree of parallelism option controls the number of processors that can be used to run a single Microsoft SQL Server statement using a parallel execution plan.  The default value for this configuration is 0 that indicates that all available processors can be used.  Though the parallelism is often beneficial for longer running queries or for queries that have complicated execution plans.  However, OLTP-centric application performance could sometimes suffer when parallel plans use more threads than the number of physical processors.  The test case now makes the optimal recommendations in the NUMA architecture.

 

 

Data Collection

MAXDOP test case performs the data collection using T-SQL script querying the system tables:  sys.dm_os_sys_info, sys.dm_os_memory_clerks and sys.configurations.

 

There are no additional configuration options for this test case. It executes against the SQL instances as configured in the Change Settings\RAP tab.  Data will be returned for this test case only if there are rule violations that occur. 

 

 

Results

The results screen for MAXDOP  reports the server name, instance name, MAXDOP Configured Value and MAXDOP Optimal value.

 

Detailed results (if applicable) can also be found in the SQL Results tab of the Issues screen
