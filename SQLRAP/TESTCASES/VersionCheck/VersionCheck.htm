Version Check

The Version Check test case verifies whether the recommended Service Pack, Cumulative Update, or hotfix for SQL Server is installed.

 

Service packs are the main delivery vehicle for fixes, security patches, and general improvements to the SQL Server system. These updates can protect you from as well as provide you with solutions to known issues. Therefore, applying service packs and hotfixes as soon as possible after thorough testing can greatly reduce your vulnerability to a wide range of issues. 

 

Additionally, because service packs often include improvements to performance, supportability, and diagnostics, having the latest service packs installed can improve response in general and can reduce the amount of time necessary to diagnose and troubleshoot an issue.   

 

Finally, support for older Microsoft® SQL Server versions and service packs could be discontinued, thus leaving your system in an unsupported configuration. Thus, it is essential to apply all service packs as soon as possible.

 

Over time, a hotfix or a Quick Fix Engineering (QFE) build is created by the development team to address specific product issues affecting certain customers. Sometimes the issue has a broad customer impact, security implications, or both. Thus, a General Distribution Release (GDR; also known as a Cumulative Update) is issued so that all customers can receive the updates.

 

After the availability of a service pack is announced, it should be applied to a test region of the environment. It should also be subjected to a comprehensive testing program that involves all the applications in the environment that use SQL Server. In addition, the tests should be conducted in such a way that a load similar to, if not equal to, the loads experienced in production are run against the test system.  Hotfixes should only be installed when specifically instructed by Microsoft Support.

 

Data Collection

The Version Check test case data is collected using a T-SQL script which queries the instance as follows:

·         VersionCheck (sql_verify_sql_version.sql):

§  Sets memory variables to the current GDR and QFE releases for SQL Server 2000 and SQL Server 2005.  Queries the value of SERVERPROPERTY (’productversion’) for the instance, extracts the build number from that value, and generates the appropriate diagnostic based on build number and platform.

There is no specific configuration necessary for this test case.  It will execute against the instance given in the Change Settings/RAP tab.

 

Results

The results screen for Version Check reports the server name, instance name, recommended build, and current build for each instance which is not running a recommended release. 

 

The SQL Results tab of the Issues view will enumerate these details as well.
