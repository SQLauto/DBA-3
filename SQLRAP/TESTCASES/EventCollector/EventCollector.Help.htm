Event Collector

Regularly reviewing the event logs is critical to ensuring the health of the environment. Although this task should be offloaded to a monitoring product for constant analysis and alerting features, administrators should still check their SQL Servers on some periodic basis. Many administrators do not do this, because they think the task is too time consuming and that there are too many false errors or errors they do not understand. Failure to remain aware of the errors reported by SQL Servers can often cause small issues to turn into major problems.

 

Data Collection

This test case collects data from the System, Application and SQL Error logs to not only flag issues, but give you a good overall idea of errors that are occurring in the system.  This test case can be configured in Change Settings\General\Event Collector where you can configure the number of days of events collected, which logs to include and other pertinent filters.   

 

The Event Collector Test Case is designed to be able to collect a summary of unique events against remote machines.  By default, it uses WMI to do the data collection unless the remote machine is windows 2000 which in that case it will use .NET.  For SQL Error log, the data is collected via a custom SQL script.

 

Results

The results from this test case are displayed in two primary sections, the RAW tab which displays all of the events that meet the filter criteria set in Change Settings and the Servers tab which allows you to see a more detailed breakout of the Events by Server and Log.

RAW

§  Server: The server the error was found on. 

§  Log: The log the error was found in.

§  Level: The severity/state level of the error (where applicable).

§  Occurrences: Number of occurrences of the event in the log for the data collection period.

§  Source: Source of the error as defined in the error or event log itself.  This can help track down where the issue may be occurring.  For SQL Error log erros, this is the generic error message – you can use this field to pivot the specific individual errors.

§  First Occurrence: The first time the error/event occurred within the data collection period (configurable)

§  Last Occurrence: The most recent occurrence of the error/event.

§  Message: The message returned with the error directly from the event/error log. For the SQL Error log, this is the more specific error message (as opposed to Source which is the generic error message).

Servers

The Servers tab provides the same information as the RAW tab, but is broken out by sub-tabs for the System, Application and SQL Server error logs
