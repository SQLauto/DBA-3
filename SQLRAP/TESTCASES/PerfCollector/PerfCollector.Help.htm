Perf Collector

The PerfCollector test casehas been created to provide a common way to gather, display and analyze performance data. The Windows built-in logman functionality is used to automate the creation, scheduling and control of performance data collections against remote servers.  Logman provides the facility of creating a counter set that will be started immediately or at a particular time in the future.  Logman is also used to query the status of performance data collection and provide feedback as to the status of performance data collection to the user at the RAPID console.

 

Once the data has been collected and has been copied to the Data Collection Machine for analysis PerfCollector leverages the logparser tool to manipulate the data that has been returned to create charts and summarize data for analysis by the rules engine within RAPID.

 

Data Collection

Data collection for the Perf Collector test case is highly involved and can be complex.  The details of the data collection for this test case is found in Appendix B of the Technical Reference.  As a summary, the data collection and test case execution takes place in two steps:  scheduling and collecting the counters and execution of the test case to perform the analysis of the data and build the results.

There are multiple configuration options for this test case:

§  Collection Interval (seconds): Set in the Change Settings\RAP\Perf Collector tab. This determines how frequently the counters are collected (default is 5 seconds)

§  Collection Time (minutes): Set in the Change Settings\RAP\Perf Collector tab. This determines the duration of the time to collect the performance counters (default is 60 minutes)

§  Collection Roll Up Interval (seconds): Set in the Change Settings\General screen.  This determines the roll up intervals of the data collected as they are to be rendered in the Results screen. 

Example:  If collection interval is 5 seconds, and the collection roll up interval is 30 seconds, the Results would use the 6 data points over each 30 second interval to calculate the min, max and average for each collection interval.

§  Rule Roll Up Interval (minutes): Set in the Change Settings\General screen.  This determines in what intervals the rules will be applied in. 

For example, if the Rule Roll Up Interval is set at 30 (default), then the recommended threshold for the counter will be checked against the max, min or average (depending on the rule) for each 30 minute interval.

 

Results

The Results for the Perf Collector test case can be very useful as it provides significant information about the performance counters that are gathered.  There are four tabs to the Results view which can be used to analyze the counters:

Charts

On this tab, you will find a default set of charts that are automatically generated each time the test case is run.  These charts plots combinations of counters that are often looked at together to see how they relate to each other or discern if there are patterns between them.  Below is a list of charts that you will find on this tab (note: all of these charts are in UTC time date format):

§  Disk I/O Chart:  There will be one of these charts created for every disk on every server targeted with the test case.  It plots the following two counters:

    • Avg Disk sec/read
    • Avg Disk Sec/write

 

  • Processor Utilization: There will be one of these charts created for every server targeted with the test case.  It plots the following five counters:

§  Processor: %Processor Time

§  Processor: %privileged time

§  Process (SQL Server): %processor time

§  System: Context Switches/sec

§  System: Processor Queue Length

 

§  Memory: There will be one of these charts created for every server targeted with the test case.  It plots the following four counters:

§  Memory: Available Mbytes

§  Memory: Free System Page Table Entries

§  Memory: pages/sec

§  Paging file: %usage peak

 

  • SQL Server Access Methods: There will be one of these charts created for every instance on every server targeted with the test case.  It plots the following five counters:

§  SQL Server statistics\Batch requests/sec

§  SQL Server Access methods\Forward records/sec

§  SQL Server Access methods\Freespace scans/sec

§  SQL Server Access methods\Workfiles/sec

§  SQL Server Access methods\worktables/sec

 

  • SQL Server Buffer Manager: There will be one of these charts created for every instance on every server targeted with the test case.  It plots the following six counters:

§  SQL Server memory manager\Total Server memory

§  SQL Server Buffer manager\Page life expectancy

§  SQL Server Buffer manager\check pints/sec

§  SQL Server Buffer manager\Lazy writes/sec

§  SQL Server Buffer manager\Free pages/sec

§  SQL Server Buffer manager\Memory grants pending/sec

 

  • SQL Server Performance: There will be one of these charts created for every instance on every server targeted with the test case.  It plots the following four counters:

§  Processor %Processor Time

§  SQL Server statistics \Batch requests/sec

§  SQL Server statistics\compilations/sec

§  SQL Server statistics\recompilations/sec

 

  • SQL Server Locking and Blocking: There will be one of these charts created for every instance on every server targeted with the test case.  It plots the following four counters:

a.     SQL Server Locks\Lock requests/sec

b.    SQL Server Locks\Lock wait time(ms)

c.     SQL Server general statistics\Processes blocked/sec

d.    SQL Server Latches\Total latch wait time(ms)

 

Server

This tab provides data about each server-level counter for each server assessed.  There are three variables options for looking at the raw data and graphs:

  • Use a combination of Server and Counter to select the counter you wish to view on the server you wish to view.
  • Rollup Type:  This allows one to display the data either by the Collection Roll up or the Rule Roll up intervals (as defined in the Data Collection section)

The data collected will be displayed on the left and the associated chart will be displayed on the right.  The associated chart will always display data in the Collection Roll Up intervals.

 

SQL Server

This tab provides data about each instance-level counter for each instance assessed.  The options for displaying the data is the same as the Server tab, only the counters are different.

 

Issues associated with each of the counters can be found in the Issues view if the counter falls outside of the recommended threshold.  These thresholds and overall recommendations for each counter is outlined in the Additional Checks and Information section below.
