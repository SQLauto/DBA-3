Pending Changes

The Pending Changes test case reports all occurrences of system configuration settings with differing run and config values.

 

Microsoft SQL Server uses system configuration settings to control certain behaviors.  Each of these settings as two values: a run value, and a config value.  The run value is the currently active value, and the config value is the value that will be applied at the next instance restart.  In general, these two values will be the same.  When they’re different, there is a pending change waiting to be applied to the system.  Leaving the system in this state for a long period of time can promote misunderstandings among staff members and, in extreme cases, system instability.

 

These are set via the sp_configure system stored procedure, and applied via the RECONFIGURE command and/or by restarting the instance (changes to certain settings requires an instance restart). 

 

Data Collection

The Pending Changes test case data is collected using a T-SQL script which queries the instance as follows:

·         SQL_SYS_CFNG_Pending_Changes.sql:

§  Queries system tables (sysconfigures, syscurconfigs, and spt_values in SQL Server 2000; sys.configurations in SQL Server 2005 and later) for configuration settings where the run value differs from the configuration value.

There are no additional configuration options for this test case. It executes against the SQL instances as configured in the Change Settings\RAP tab.  Data will be returned for this test case only if there are rule violations that occur. 

 

Results

The results screen for Pending Changes reports the server name, instance name, configuration name, configured value, and run value for each configuration setting with differing configured value and run value.

 

Detailed results (if applicable) can also be found in the SQL Results tab of the Issues screen.
