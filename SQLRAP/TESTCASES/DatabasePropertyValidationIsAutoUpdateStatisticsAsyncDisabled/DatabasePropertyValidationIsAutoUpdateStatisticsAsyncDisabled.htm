IsAutoUpdateStatsAsync

The DatabasePropertyValidation - IsAutoUpdateStatsAsync test case reports all user databases which have enabled the AUTO_UPDATE_STATISTICS_ASYNC database property.  This test case only generates results for SQL Server 2005 and greater instances.

 

AUTO_UPDATE_STATISTICS_ASYNC provides asynchronous statistics updating. When this option is set to ON, queries do not wait for the statistics to be updated before compiling. The query and any other concurrent queries compile immediately by using the existing out-of-date statistics. Because there is no delay as a result of waiting for updated statistics, query response times are predictable. However, the out-of-date statistics may cause the query optimizer to choose a less efficient query plan.

 

You should consider changing the AUTO_UPDATE_STATISTICS_ASYNC option to ON when the following characteristics apply to your application:

·         Client request timeouts caused by one or more queries waiting for updated statistics have been experienced.

·         Predictable query response times are required, even at the expense of occasionally running queries with less efficient query plans because of outdated statistics.

Data Collection

The DatabasePropertyValidation - IsAutoclose test case data is collected using a T-SQL script which queries the instance as follows:

·         DatabasePropertyValidation – IsAutoUpdateStatisAsync (sql_is_auto_update_statistics_async_disabled.sql):

§  Reports a list of databases for which the column master.sys.databases.is_auto_update_stats_async_on = 1.

There are no additional configuration options for this test case. It executes against the SQL instances as configured in the Change Settings\RAP tab.

 

Results

Upon running this test case, there will be data in the Results screen only if this condition exists. The results screen for DatabasePropertyValidation - IsAutoUpdateStatisticsAsyncDisabled reports the server name, instance name, database name, and “Async Auto Update Stats Enabled” for each database on the instance for which the AUTO_UPDATE_STATISTICS_ASYNC database property is on.

 

In the Issues view, you can use the SQL Results tab to view this data as well for this issue.
