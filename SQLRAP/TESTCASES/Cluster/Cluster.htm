Cluster

The Cluster test case displays significant information about your cluster configuration, dependencies and nodes.  It will also evaluate that data against best practices and raise corresponding issues. Proper configuration and maintenance of your cluster is a key component of a robust High Availability Strategy.

 

Data Collection

Data is collected via a variety of WMI and registry queries to provide the proper data.  For Windows 2003, this is primary done via WMI and a blend of WMI and registry settings for Windows 2000.

There aren’t many configuration options for this test case. It runs against the windows cluster name as set in the Change Settings/RAP/Targets portion of the toolset.  If there is no cluster set to be targeted, there is no need to run this test case.

 

Results

In the Results control, you can easily see key data that is collected for the cluster. There are two primary tabs that reveal this data, Cluster Information and Node Information.  The data and behavior for each of these is provided as follows:

Cluster Information

§  Cluster Data Pane
This is the top pane in this tab and simply contains the Cluster name(s) as entered in the Change Settings/RAP/Targets portion of the toolset along with the time of the last execution of the test case against that cluster.  The detailed data in Resource Information pane is based on what Cluster name is selected in this pane. 

§  Resource Information Pane
This pane contains the list of resources for the cluster selected in the Cluster Data Pane. Selecting a resource in this screen will populate the Resource Dependencies screen with which resources it is dependent on. The data provided is:

§  Group Name: The name of the group the resource is in

§  Name: Name of the resource

§  Restart Action: Action taken if the resource fails.  Identified as number 0,1 or 2 based on the following table:

Name

Value

Description

ClusterResourceDontRestart

0

Do not restart the resource after a failure.

ClusterResourceRestartNoNotify

1

Restart the resource after a failure. If the resource exceeds its restart threshold within its restart period, do not attempt to failover the group to another node in the cluster.

ClusterResourceRestartNotify

2

Restart the resource after a failure. If the resource exceeds its restart threshold within its restart period, attempt to fail over the group to another node in the cluster. This is the default setting.

 

§  Type: The type of the resource created.  Whether it be IP Address, SQL Server, File Share, Physical Disk etc.

§  Resource Dependencies
Upon selecting a Resource from the Resource Information pane, this pane populates with the appropriate resource dependencies.  Example: if you select Cluster IP Address in the Resources Information pane, and the Resource Dependencies populate with Resource Dependent On = Quorum Disk, then the Cluster IP address is dependent on the Quorum Disk.

 

Node Information

This tab contains two panes one with a list of clusters and another with a list of details about that cluster and sub-tabs. 

§  Server Information tab

§  Server Name: The server name for each node in the cluster.

§  DNSHostName: This informational data point is gathered via the registry key below.  No issues or specific rules are raised from it, but it can be useful information about the server.

§  Domain Role:  Role of a computer in an assigned domain workgroup. A domain workgroup is a collection of computers on the same network.

0 (0x0)

Standalone Workstation

1 (0x1)

Member Workstation

2 (0x2)

Standalone Server

3 (0x3)

Member Server

4 (0x4)

Backup Domain Controller

5 (0x5)

Primary Domain Controller

 

§  Manufacturer: The name of a computer manufacturer.

§  Model: Product name that a manufacturer gives to a computer.

§  Number of Processors: Number of physical processors currently available on a system. This is the number of enabled processors for a system, which does not include the disabled processors. If a computer system has two physical processors each containing two logical processors, then the value of NumberOfProcessors is 2 and NumberOfLogicalProcessors is 4. The processors may be multicore or they may be hyperthreading processors. For more information, see Remarks.

Windows Server 2003, Windows XP, and Windows 2000:  Because the NumberOfLogicalProcessors property is not available, NumberOfProcessors indicates the number of logical processors available in the system. In the case of a computer system that has two physical processors each containing two logical processors, the value of NumberOfProcessors is 4

§  PartOfDomain: If True, the computer is part of a domain. If the value is NULL, the computer is not in a domain or the status is unknown. If you unjoin the computer from a domain, the value becomes false.

§  SystemStartupDelay: Time to delay before starting the Windows operating system—in seconds. Beginning with Windows Vista, SystemStartupDelay is obsolete because Boot.ini is not used to configure system startup. Instead, use the BCD classes supplied by the Boot Configuration Data (BCD) WMI provider or the Bcdedit command.

§  SystemStartupOptions: List of the options for starting up the coumputer system running Windows. This property must have a value. Beginning with Windows Vista, SystemStartupOptions is obsolete because Boot.ini is not used to configure system startup. Instead, use the BCD classes supplied by the Boot Configuration Data (BCD) WMI provider or the Bcdedit command.

§  TotalPhysicalMemory: Total size of physical memory. Be aware that, under some circumstances, this property may not return an accurate value for the physical memory. For example, it is not accurate if the BIOS is using some of the physical memory. (in Bytes)

§  Processor Make: Name of the processor manufacturer

§  OS Information tab

§  Boot Device: Name of the disk drive from which the Windows operating system starts.

§  Build Number: Build number of an operating system. It can be used for more precise version information than product release version numbers.

§  Build Type:  Type of build used for an operating system.

§  Caption:  Short description of the object—a one-line string. The string includes the operating system version. For example, "Microsoft Windows XP Professional Version = 5.1.2500". This property can be localized.

§  Cluster Log File Size: Size of the cluster log in MB

§  LocalDateTime:  Operating system version of the local date and time-of-day.

§  MaxProcessMemorySize: Maximum number, in kilobytes, of memory that can be allocated to a process. For operating systems with no virtual memory, typically this value is equal to the total amount of physical memory minus the memory used by the BIOS and the operating system. For some operating systems, this value may be infinity, in which case 0 (zero) should be entered. In other cases, this value could be a constant, for example, 2G or 4G.

§   PAEEnabled: If True, the physical address extensions (PAE) are enabled by the operating system running on Intel processors. PAE allows applications to address more than 4 GB of physical memory. When PAE is enabled, the operating system uses three-level linear address translation rather than two-level. Providing more physical memory to an application reduces the need to swap memory to the page file and increases performance. To enable, PAE, use the "/PAE" switch in the Boot.ini file. For more information about the Physical Address Extension feature, see http://go.microsoft.com/fwlink?LinkID=45912.

§  ServicePackMajorVersion: Major version number of the service pack installed on the computer system. If no service pack has been installed, the value is 0 (zero).

§  ServicePackMinorVersion: Minor version number of the service pack installed on the computer system. If no service pack has been installed, the value is 0 (zero).

§  SystemDevice: Physical disk partition on which the operating system is installed.

§  SystemDirectory: System directory of the operating system.

§  SystemDrive: Letter of the disk drive on which the operating system resides. Example: "C:"

§  Version: Version number of the operating system.

§  WindowsDirectory: Windows directory of the operating system.
