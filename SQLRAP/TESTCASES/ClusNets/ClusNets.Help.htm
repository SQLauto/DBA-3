Cluster and Node Network - General

DESCRIPTION:

This check confirms the configuration of the network interfaces in the cluster on a node by node basis. Furthermore it checks the heartbeat/private network configuration.

DETAILS:

·         Cluster node uses NIC teaming on the Private network (network designated to be “Internal Cluster Communications only (private network)”)

·         Server(s) has Networks that are not defined for cluster use

·         Server(s)'s fixed setting for NIC Speed, NIC Duplex, and network switch are not the same

·         Server does not have dynamic updates enabled for all necessary DNS record types on Public Interface

·         Server has a misconfigured Default Gateway on the Public Interface

·         Server has multiple Default Gateways defined

·         DNS name resolution failed for Cluster Name {1}. Check your DNS configuration.

·         DNS name resolution failed for Node Name %NODES%. Check your DNS configuration.

·         Server(s) has "Client For Microsoft Networks" enabled on the Private Network Interface while it is a shared quorum model

·         Server(s) does not have "Client For Microsoft Networks" enabled on the Private Network Interface while it is a majority node set quorum model

·         Server(s) has "File and Print Sharing" enabled on the Private Network Interface while it is a shared quorum model

·         Server(s) does not "File and Print Sharing" enabled on the Private Network Interface while it is a majority node set

·         Server(s) has "Client For Microsoft Networks" disabled on the Public Network Interface

·         Server(s) has "File and Print Sharing" disabled on the Public Network Interface

·         Server(s) has NetBT Bound enabled on the Private Interface

·         Server(s) does not have NetBT Bound enabled on the Public Interface

·         Server(s) has LMHOSTS enabled when it is not the only method of name resolution

·         Server(s) has DisableDHCPMediaSense value in the registry

·         Server has IP Addresses dynamically assigned on the network interfaces

·         Server(s) has server configured as either RAS client or server

·         Server(s) has 3rd Party Service Winsock providers installed

·         Server(s) has IPSec enabled with Filters defined for the Public Interface

·         Server(s) has IPSec enabled with Filters defined for the Private Interface

·         Server(s) has the Firewall Service enabled on Public or Private Networks

·         Server(s) has multicast enabled in a cluster with than 3 or more nodes

·         Cluster node has no physical redundancy on the Public Network (networks designed to be “All communications (mixed mode)” or “Client access only (public network)”)

·         Server(s) has NIC teaming installed on those interfaces designated as Private Networks (heartbeat)

·         Server has Public Network not bound as Primary Network Service Interface

·         Server has Private Network bound as Primary Network Service Interface

·         Cluster does not have a network whose role is configured as Internal cluster communications only (private network)

·         Cluster node has no physical redundancy on the Public Network (networks designed to be “All communications (mixed mode)” or “Client access only (public network)”)

·         Server(s) does not have the Private Network used for Internal Communications Only set as the highest priority network in the cluster

·         IGMPVersion has been set on the Cluster Node and configured to a value lower than the Default of 4. This is necessary on a Cluster Node using Heartbeat communication

·         Server does not have the Public NIC bound as Secondary Priority

·         Server does not have the Private NIC Bound as Primary Priority

·         Server has 3rd Party Name Service Winsock installed

·         Server(s) does not have a valid DNS Server defined for Name Resolution on the Public Interface
