SA Owned Jobs

 

The sysadmin Owned Jobs test case reports all jobs or job steps running under a sysadmin credential.

 

Jobs owned by a logon account that is a member of the sysadmin fixed server role will be run under the security context of the Microsoft® SQL Server Agent account.  Jobs that are not run under a member of the sysadmin fixed server role will be run under the domain user account for the SQL Server Agent proxy account.  It is generally desirable to run a particular job under the credential with the lowest possible privilege necessary to complete the tasks.

 

 

Data Collection

The sysadmin Owned Jobs test case data is collected using a T-SQL script which queries the system tables as follows:

·         sysadmin Owned Jobs (SQL_jobs_owned_by_sa.sql):

o    On SQL Server 2000, all jobs are reported where msdb.dbo.sysjobs.owner_sid is a member of the sysadmin role, as well as all jobsteps where msdb.dbo.sysjobsteps.database_user_name is a member of the sysadmin role and msdb.dbo.sysjob.owner_sid for the associated job is not a member of sysadmin.
Because of the security model changes in SQL Server 2005, the logic in that branch also considers non-TSQL jobsteps running under proxy accounts as well as all of the SQL Server 2000 scenarios.

 

There are no additional configuration options for this test case. It executes against the SQL instances as configured in the Change Settings\RAP tab.

 

Results

Upon running this test case, there will be data in the Results screen only if this condition exists. The results screen for sysadmin Owned Jobs reports the server name, instance name, run context, job name, and job step name for all jobs and job steps running under a credential with sysadmin privilege.

 

In the Issues view, you can use the SQL Results tab to view this data as well for this issue.
